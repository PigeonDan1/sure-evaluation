# Pipeline Catalog

This directory contains a machine-readable catalog of every metric pipeline that `sure-eval metric describe` can produce.

## File: `pipeline_catalog.jsonl`

Each line is one JSON object describing a single metric evaluation route. The file is generated by running `sure-eval metric describe` for every supported task + language + metric combination.

### Schema

| Field | Type | Description |
|:------|:-----|:------------|
| `task` | string | Canonical task name (e.g. `ASR`, `TTS`) |
| `task_alias` | string | CLI task argument (e.g. `asr`, `tts`) |
| `language` | string | Language profile, or `n/a` for language-independent tasks |
| `metric` | string | CLI metric/route selector. Some selectors, such as `cer_canonical`, select an alternate normalization route for an existing metric family. |
| `pipeline_id` | string | Resolved pipeline ID |
| `nodes` | list | Ordered list of selected node IDs |
| `required_roles` | list | Input roles you must provide to `sure-eval metric run` |
| `optional_roles` | list | Input roles you may optionally provide |

### Example row

```json
{
  "task": "ASR",
  "task_alias": "asr",
  "language": "zh",
  "metric": "cer",
  "pipeline_id": "asr.zh.cer.wetext_zh_itn.wenet_cer",
  "nodes": ["normalization/wetext_norm", "scoring/wenet_cer"],
  "required_roles": ["hyp", "ref"],
  "optional_roles": []
}
```

## How to read the catalog

Find the metric you want to run, then look at `nodes` and `required_roles`:

- `nodes` tells you which evaluation nodes are executed.
- `required_roles` tells you which CLI arguments or Python API fields are mandatory.

For example, `s2tt.zh.xcomet_xl` requires `src`, `hyp`, and `ref`:

```json
{
  "task": "S2TT",
  "task_alias": "s2tt",
  "language": "zh",
  "metric": "xcomet_xl",
  "pipeline_id": "s2tt.zh.xcomet_xl.xcomet_xl",
  "nodes": ["scoring/xcomet_xl"],
  "required_roles": ["src", "hyp", "ref"],
  "optional_roles": []
}
```

## How to regenerate

```bash
python scripts/generate_pipeline_catalog.py
```

The script runs `describe_pipeline` for every supported combination and writes `docs/pipeline_catalog.jsonl`.

## Notes

- Multi-metric audio routes (e.g. `tts_cer + sim/wavlm-large + dnsmos` or `si-sdr + pesq + dnsmos`) use the same `pipeline_id` but include all selected nodes.
- Node-local environments are required for heavy nodes such as `scoring/xcomet_xl`, `scoring/dnsmos`, and `transcription/whisper_large_v3`. Optional pip packages are required for nodes such as `scoring/stoi` and `scoring/pesq`. Use `sure-eval env setup --node <node>` to prepare them.
